FoilTrans

 

FoilTrans ’transforms’ basic Cl, Cd, Cm data to X-Plane foil files (.afl files). It does this by statistical analysis and best fit curves. It is particularly suitable for polars made from Martin Hepperle’s JavaFoil, http://www.mh-aerotools.de/airfoils/javafoil.htm or Xfoil, http://raphael.mit.edu/xfoil/ because it reads data files made by these programs, but can also be used for generic data. Downloads, manuals etc for those programs can be found at the sites mentioned above.

 

After installation you will need to install the LabVIEW runtime environment for LabVIEW 7.0 (20MB download). ftp.ni.com/support/labview/runtime/windows/7.0/LVRunTimeEng.exe

The first thing to do is to press Init->Set Default XP Background File. This initiates the data and loads a ‘default’ .afl foil. Your view will look like the one in Figure 2.

 

Figure 2 after initiation

 

 

Then you load your data file saved with JavaFoil, Xfoil or a generic one. The data has to start on –20 degree, end on +20 degree and should have a top Cl before +20 degree[1].

 

You do this by File->Load Foil Data->xxx, where xxx is one of several options. The data is either smoothed with a cubic spline, or interpolated linearly. I found that good results are obtained when using JavaFoil with 1 or 2 degrees steps for the data and cubic spline smoothing. For Xfoil it is probably better to save the data at 0.1 degree intervals and use either splines or linear interpolation. You just have to try different options and see what looks best. When using generic data, the data needs to be formatted as shown below. Alpha1 must to be –20 degrees and alphaN must be +20 degrees, and the file must contain only numbers. Between –20 and +20 degrees you can have as many points as you like, just as long as you have max Cl before +20 degree and min Cl after –20 degree.

Now you have to do one thing manually, and that is to prescribe the Cm curve. You do this by first pressing the ‘Scale’ slider so that the view is from –20 to +20 degrees. Your Cm curve should now look something like Figure 4. On the Cm curve are two cursors marked as a cross. The idea is to move those cursors with the mouse to fit the green curve to the red curve (the red curve is the loaded data). If you can’t find the cursors, then press the ‘Centre Cm Cursors’ and they will go to 0,0. The green lines will follow the cursors.

Then you set your Re number, Thickness and Drag Div Mach to appropriate numbers for your foil. This is done in the upper left corner.

 

Press the ‘Modded foil’ tab (should be all zeroes right now).

 

Then press Modify->Calculate parameters for XP foil format. This will calculate all the parameters needed (intercept, slope linear range etc) and you have a valid XPlane foil that is the best statistical approximation to your data, considering that your data is fairly good.

 

Now you can press Modify->Extend foil data to –180 +180 and press File->Save modded file and you have a valid X-Plane .afl file that you can use.

 

In Airfoil-Maker you can change one number, then change it back so that AM ‘builds’ an airfoil based on the parameters and extends it to –180 and +180 degrees based on it’s own functions using the parameters from FoilTrans. For compatibility reasons this will be the best choice and virtually nothing is changed in the range ‘–linear range’ to +20 degrees for the Cl curve (for normal flying you won’t notice any difference at all). However, due to the parametric curves in Airfoil-Maker you will lose some details, especially for the Cd and Cm curves outside the linear range.

 

 

Other options and things

 

AM seems to extend the data to –180 and 180 degrees by multiplying Cl with 0.5, Cd by 4.0 and Cm by 0.0. In FoilTrans you can change this by setting your own values in the ‘Extend data’ area. For instance when setting all three parameters to 1.0, your foil will have the same characteristics flying backwards and forwards. You can save the plots as bmp files, paste your foil data directly into the background XP foil, and study the data more closely by using the graph tools just left of the graphs.

 

In the install directory there is an ini file with one value. By changing it to True the program will use the local decimal point (comma in Europe), otherwise it will use a decimal dot. It will read and write correctly (I hope) no matter what you use.

 

 

Calculations of data

I found that with Airfoil-Maker in XP you can reproduce the foil data quite accurate in the range -20 to +20 degree in general. This range is basically where all the parameters are set. The Cl curve is defined and thus, very accurate in the range –linear range to +20 degree. The Cd curve is defined from Cd min to +10 degree, while the Cm curve is fairly accurate in the range of the linear range of Cl. Outside those ranges the accuracy decreases. The good point with the Airfoil-Maker approach is that the data gets effectively smoothed as they should be, but in some cases it will be a bit restrictive.

 

In FoilTrans the parameters are found using best-fit statistics on the data. In the range –20 to +20 degree the step size is 0.1 degree (-20.0 –19.9 –19.8 ,, +19.9 +20.0). When reading in data, FoilTrans just reads line by line and take whatever it gets. The data is then interpolated in a step size of 0.1 degree using linear interpolation or cubic splines.

 

The intercept, slope and linear range are found using linear best-fit approximation. This is done stepwise by starting at zero aoa and increasing the range 0.1 degree in both directions and calculating the best fit linear line and its mean square error, mse. When the mse is larger than 1e-5 it stops. The intercept, slope and linear range therefore have an mse of 1e-5. In addition the max range is 10 degree and min range is 1 degree.

 

The max and min values are found simply by finding them in the data. The ‘power’ data values are found using the Levenberg-Marquardt algorithm on the data with a  function. b is then found to be the best fit ‘power’ value while a is found from the start and end values of the piece of curve.

 

 

 

Foil Calculation Software

I have only used Xfoil and JavaFoil, and I found that JavaFoil is better for general use for high Re numbers. JavaFoil seems to give data that is more in accordance with wind tunnel tests than Xfoil does (often spot on), and it also seems to have a better stall model (the Eppler stall model). Xfoil is presumed to be more advanced, but that is only relevant for low Re numbers when laminar separation bobbles are important as far as I can tell. Xfoil does have better foil editing tools.
